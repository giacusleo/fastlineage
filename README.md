# FastLineage

FastLineage is a lightweight VS Code extension for exploring dbt lineage directly inside the editor panel.

## What it does

- Reads `dbt_project.yml`, scans configured model/seed/snapshot paths, and builds a lineage graph from `ref()` and `source()` calls.
- Focuses the graph around the active model so upstream and downstream context stay readable.
- Lets you pan, zoom, re-root, and jump from graph nodes back into the corresponding file.

## Current scope

FastLineage is intentionally small and fast. Today it is SQL-driven and does not require `manifest.json`.

It is designed to work against real dbt workspaces, whether they contain one dbt project or several sibling/local-package projects:

- Respects `model-paths`, `seed-paths`, and `snapshot-paths` from `dbt_project.yml`.
- Reads relation metadata from standard dbt schema YAML files, including files that document multiple models, seeds, snapshots, or sources.
- Resolves the active dbt project from the open file when possible, then discovers other `dbt_project.yml` files in the workspace for multi-project lineage.
- Resolves unqualified refs against the current project first, then declared local package projects from `packages.yml` or `dependencies.yml`.
- Treats package-qualified refs as external nodes unless the referenced package is also present in the open workspace.

## Install locally

If you want to try the extension without publishing it:

1. Open the repository's GitHub Releases page.
2. Download the latest `fastlineage-*.vsix` asset.
3. In VS Code or Cursor, open Extensions and choose "Install from VSIX...".

If you want to build it yourself instead:

1. Clone the repo.
2. Run `npm install`.
3. Run `npm run package`.
4. Install the generated `.vsix` file via "Install from VSIX...".

## Develop

1. Run `npm install`.
2. Run `npm run compile`.
3. Press `F5` to launch an Extension Development Host.
4. In the Extension Development Host, open any dbt project folder or multi-project dbt workspace and run FastLineage.

## Repo guide

- `src/` contains the extension source.
- `out/` is generated by `npm run compile` and is not tracked.
- `media/` contains the webview assets.
- `docs/` contains repository-level docs such as packaging and publishing notes.
